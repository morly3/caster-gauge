# Caster Gauge

GitHub Pages: https://morly3.github.io/caster-gauge/

Version: v0.5.1
Updated: 2026-07-17

スマートフォンをホイール用アライメントゲージに固定し、重力ベクトルの静止サンプルからステアリング回転軸を推定してキャスター角を測る GitHub Pages 向けの単一 HTML アプリです。

## 使い方

1. ブラウザで `index.html` を開き、必要ならセンサー許可を与えます。
2. スマートフォンをゲージに固定します。測定中は必ず画面がホイールの外側を向くように立てます。
3. エンジン停止中は「標準」、エンジン稼働中は振動量に応じて「振動対応・弱／強」を選びます。
4. 直進状態で「直進基準取得」を押します。選択した判定時間にわたって平均姿勢が安定すると、直進基準が自動記録されます。
5. ステアリングを左方向から右方向まで、間隔を空けて複数位置へ動かします。直進基準の記録後は、安定状態になるたびに最大 2 回取得し、一つの静止位置としてまとめます。
6. 自動追加されない位置だけ「手動で取得」を押します。
7. 5 位置以上になると、測定点が追加されるたびに推定結果が自動更新されます。
8. キャスター角、キングピン角相当、残差、測定品質を確認します。
9. 必要に応じて CSV を保存します。

対応ブラウザでは、測定中に Screen Wake Lock API で画面スリープを抑止します。未対応ブラウザでは端末側の自動ロック設定を長めにしてください。

自動取得は静止状態だけで動作します。同じ静止状態では 2 回で停止し、一度動かしてから次の静止状態になると、次の静止位置としてまた最大 2 回取得します。画面の「位置 / 取得」は、平面フィットに使う独立した静止位置数と、生の取得回数を表します。

## 静止判定レベル

- `標準（直近1.2秒を判定・平均）`: 直近1.2秒間のセンサーデータを静止判定と重力ベクトル算出に使う、従来相当の通常測定です。
- `振動対応・弱（直近3秒を判定・平均）`: 直近3秒間のセンサーデータを静止判定と重力ベクトル算出に使い、軽いアイドリング振動を複数ブロックの平均で相殺します。
- `振動対応・強（直近5秒を判定・平均）`: 直近5秒間のセンサーデータを静止判定と重力ベクトル算出に使い、より大きい振動を長い平均時間とトリム平均で弱めます。

この秒数は、各取得で参照するローリングデータ窓の長さです。静止位置へ移動した直後は、その時間幅のデータが安定して蓄積されてから取得します。同じ位置の2回目は、別に設定したクールダウン後に更新されたローリング窓から取得します。

振動対応では、ローリング保存した3～5秒のデータを375～500msの時間ブロックへ分割し、各ブロックの平均重力ベクトルを求めます。短周期の往復振動はブロック平均で弱め、さらに極端なブロックを除くトリム平均から測定ベクトルを作ります。一方、ブロック平均の角度RMS、前半と後半の方向差、平均重力値の変動を監視するため、振動しながらゆっくり転舵している状態は静止として受理しません。

判定レベルを変更すると、既存の測定点は残したまま、判定用の直近データだけをリセットして再蓄積します。

## 測定方式

`DeviceOrientationEvent` の `alpha / beta / gamma` はキャスター計算に使いません。主入力は `DeviceMotionEvent.accelerationIncludingGravity` の `x / y / z` です。

各ステア位置で選択レベルに応じて1.2～5秒の重力ベクトルをサンプリングし、多段平均したベクトルを正規化して保存します。同じ安定状態で取得した最大 2 回分は、一つのセッション代表ベクトルへ平均してから推定に使います。複数位置の代表ベクトル群は、理想的にはステアリング回転軸に垂直な平面上に並びます。そのため、点群の共分散行列を作り、最小固有値に対応する固有ベクトルをステアリング回転軸として推定します。

同じ姿勢付近だけが過大に効かないよう角度密度から基礎重みを計算し、その範囲を 0.5～2 倍に制限します。さらに1セッションずつ除外した候補から頑健な初期平面を選び、Tukey biweight による反復再重み付け平面フィットを行います。これにより孤立した外れ値を0重みにでき、推定軸を強く引っ張らないようにします。正常位置が5未満になる場合だけHuber重みへ退避します。保持上限を超えた場合は、同じセッションの重複取得、平面残差が大きい点、密集領域の点を優先して破棄します。

キャスター角の推定幅は、静止セッションを一つずつ除外して再計算する delete-one ジャックナイフ標準誤差 `SE` と、おおよその `±95%` 幅として表示します。同じ静止位置の2取得を独立点として数えず、従来不足していたジャックナイフ係数も適用します。

画面の「安定度」は、`SE`、ロバスト RMS 残差、転舵による重力ベクトルの角度幅、平面法線を識別する固有値分離、外れ値数から判定します。旧版の「最大固有値÷中間固有値」による転舵範囲判定は廃止しました。`安定` はセッションを一つ抜いても推定角がほとんど変わらず、転舵範囲と平面形状も十分な状態です。

この方式では、転舵角、キャンバー角、ジャイロ積分角、`DeviceOrientation` の方位情報をキャスター計算の入力値として使用しません。また、車輪軸上の仮想 3D 座標点を作成したり、三つの仮想点だけを通る平面を求めたりしません。5 点以上の実測重力ベクトルに対して、密度重み付きの最小二乗平面フィットを行います。

詳しい技術説明は [docs/measurement-algorithm.md](docs/measurement-algorithm.md) に記録しています。

キャスター角は、推定軸の鉛直成分と車両前後方向成分から計算します。

```text
caster = atan2(dot(axis, forward), dot(axis, verticalUp))
```

キングピン角相当は参考値として絶対値表示します。

```text
sai = atan2(dot(axis, lateral), dot(axis, verticalUp))
```

## 取り付け設定

手動で「ホイール面法線」やスマホ軸を選ぶ必要はありません。測定時は、画面がホイール外側を向く取り付けに固定します。

キャスター角とキングピン角相当は絶対値表示です。左右輪指定は不要です。ゲージ取り付け向きによって結果が不自然な場合だけ、詳細設定として前方向反転を調整します。

## CSV

CSV には各取得のセッションID、生平均、正規化ベクトル、サンプル数、標準偏差、静止判定レベル、取得時間、平均ブロック数、ブロック角度RMS、姿勢ドリフト、ロバストフィット重みを保存します。末尾にキャスター角、推定軸、残差、固有値、静止位置数、角度範囲、平面分離比、外れ値数、ジャックナイフ標準誤差と品質も出力します。

## デバッグ

`?debug=1` を付けて開くと、実センサーなしでシミュレーションデータを追加できます。`?debug=auto` では読み込み時に自動実行します。`?debug=vibration` では12Hzの模擬振動を受理し、同時に2度移動するデータを除外できるか確認します。

## 動作条件

- HTTPS 配信またはローカルファイル
- `DeviceMotionEvent.accelerationIncludingGravity` 対応ブラウザ
- iOS ではセンサー許可が必要です
