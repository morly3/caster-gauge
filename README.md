# Caster Gauge

GitHub Pages: https://morly3.github.io/caster-gauge/

Version: v0.3.0  
Updated: 2026-07-16

スマートフォンをホイール用アライメントゲージに固定し、重力ベクトルの静止サンプルからステアリング回転軸を推定してキャスター角を測る GitHub Pages 向けの単一 HTML アプリです。

## 使い方

1. ブラウザで `index.html` を開き、必要ならセンサー許可を与えます。
2. スマートフォンをゲージに固定します。測定中は必ず画面がホイールの外側を向くように立てます。
3. 直進状態で「直進基準を記録」を押します。
4. ステアリングを左方向から右方向まで、間隔を空けて複数位置へ動かします。直進基準の記録後は、車体の揺れが収まったタイミングで測定点が自動追加され続けます。
5. 自動追加されない位置だけ「手動で測定点追加」を押します。
6. 5 点以上になると、測定点が追加されるたびに推定結果が自動更新されます。
7. キャスター角、ステア軸左右傾斜、残差、測定品質を確認します。
8. 必要に応じて CSV を保存します。

## 測定方式

`DeviceOrientationEvent` の `alpha / beta / gamma` はキャスター計算に使いません。主入力は `DeviceMotionEvent.accelerationIncludingGravity` の `x / y / z` です。

各ステア位置で静止中の重力ベクトルを約 1.2 秒サンプリングし、平均ベクトルを正規化して 1 点として保存します。複数点の重力ベクトル群は、理想的にはステアリング回転軸に垂直な平面上に並びます。そのため、点群の共分散行列を作り、最小固有値に対応する固有ベクトルをステアリング回転軸として推定します。

同じ姿勢付近で多く点を取った場合にその領域だけが過大に効かないよう、測定点同士の角度的な密度から重みを計算し、密な領域の点は軽く、粗い領域の点は相対的に重くして平面フィットします。キャスター角の推定幅は、1 点ずつ除外した leave-one-out 推定のばらつきから `σ` とおおよその `±95%` 幅として表示します。

画面の「安定度」は `σ` と RMS 残差から判定します。`安定` は点を 1 点ずつ抜いても推定角がほとんど変わらない状態、`要確認` は使用可能だが点の追加や固定確認を推奨する状態、`不安定` は揺れ・固定・転舵範囲を見直すべき状態です。

この方式では、転舵角、キャンバー角、ジャイロ積分角、`DeviceOrientation` の方位情報をキャスター計算の入力値として使用しません。また、車輪軸上の仮想 3D 座標点を作成したり、三つの仮想点だけを通る平面を求めたりしません。5 点以上の実測重力ベクトルに対して、密度重み付きの最小二乗平面フィットを行います。

詳しい技術説明は [docs/measurement-algorithm.md](docs/measurement-algorithm.md) に記録しています。

キャスター角は、推定軸の鉛直成分と車両前後方向成分から計算します。

```text
caster = atan2(dot(axis, forward), dot(axis, verticalUp))
```

SAI 相当の左右傾斜は参考値として表示します。

```text
sai = atan2(dot(axis, lateral), dot(axis, verticalUp))
```

## 取り付け設定

手動で「ホイール面法線」やスマホ軸を選ぶ必要はありません。測定時は、画面がホイール外側を向く取り付けに固定します。

左右前輪の違い、またはゲージ取り付け向きの違いに合わせて、測定対象と前方向反転だけを調整します。一般的な正キャスターが正値になるよう、結果画面の符号説明を確認してください。

## CSV

CSV には各測定点の生平均、正規化ベクトル、サンプル数、標準偏差、使用可否、除外理由を保存します。末尾にキャスター角、推定軸、鉛直方向、左右方向、前方向、残差、固有値、品質も出力します。

## デバッグ

`?debug=1` を付けて開くと、実センサーなしでシミュレーションデータを追加できます。真値キャスター 5 度、左右傾斜 2 度相当の点群から復元確認を行うための機能です。

## 動作条件

- HTTPS 配信またはローカルファイル
- `DeviceMotionEvent.accelerationIncludingGravity` 対応ブラウザ
- iOS ではセンサー許可が必要です
